# Монте-Карло оценка девелоперского проекта: NPV и корреляция рисков

Интерактивный дашборд для оценки влияния строительных, рыночных и
регуляторных рисков на чистую приведённую стоимость (NPV) девелоперского
проекта методом Монте-Карло.

Приложение является **прикладным продолжением курсовой работы** по дисциплине
«Инвестиционная оценка»: теоретическая модель совместного распределения рисков
превращена в готовый инструмент инвестиционного анализа.

---

## Для кого этот инструмент

Dashboard предназначен для девелоперов, инвесторов, банков проектного
финансирования, оценщиков и консультантов, которым необходимо оценить не только
базовую доходность проекта, но и распределение возможных исходов с учётом
совместного наступления строительных, рыночных и регуляторных рисков. В учебном
контексте модель демонстрирует практическое применение метода Монте-Карло в
инвестиционной оценке.

**Практический смысл:** модель позволяет перейти от вопроса «чему равна NPV
проекта?» к вопросу «с какой вероятностью проект создаст или разрушит
стоимость?».

---

## Возможности

- Расчёт детерминированной (базовой) NPV проекта по заданным параметрам.
- Симуляция Монте-Карло (до 50 000 итераций) с пятью факторами риска.
- Генерация коррелированных факторов через **гауссову копулу** (разложение
  Холецкого).
- Сравнение двух моделей — с **независимыми** и **коррелированными** рисками.
- Показатели риска: средняя/медианная NPV, стандартное отклонение, VaR (5%),
  Expected Shortfall, вероятность убытка, перцентили.
- Анализ чувствительности NPV к факторам риска (торнадо-диаграмма).
- Редактируемая корреляционная матрица с автоматической проверкой на
  положительную определённость.
- Автоматический текстовый инвестиционный вывод.

## Установка и локальный запуск

Требуется Python 3.9+.

```bash
# 1. перейти в папку проекта
cd monte-carlo-developer-dashboard

# 2. (опционально) создать виртуальное окружение
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# 3. установить зависимости
pip install -r requirements.txt

# 4. запустить дашборд
streamlit run app.py
```

После запуска приложение откроется в браузере по адресу `http://localhost:8501`.

## Какие параметры можно менять

На боковой панели:

- начальные инвестиции, базовая выручка (GDV), строительные и операционные
  расходы;
- ставка дисконтирования, горизонт проекта;
- число итераций и зерно генератора (для воспроизводимости).

В основной области:

- параметры предельных распределений всех пяти факторов риска (минимум / мода /
  максимум, вероятности задержки);
- коэффициенты корреляционной матрицы (редактируются в таблице).

## Методология (кратко)

NPV рассчитывается дисконтированием денежных потоков проекта. Пять факторов
риска (строительные затраты, цена продажи, задержка, регуляторные расходы, темп
продаж) задаются треугольными и дискретным распределениями. Коррелированные
значения генерируются гауссовой копулой: из независимых нормальных величин через
множитель Холецкого корреляционной матрицы строятся коррелированные нормальные,
которые переводятся в равномерные и далее — в значения факторов. По 10 000+
реализациям оцениваются распределение NPV и показатели риска (VaR, ES,
вероятность убытка).

---

## Краткая логика модели

1. Пользователь задаёт базовые параметры проекта (инвестиции, выручка, затраты,
   ставка, горизонт) и параметры пяти факторов риска.
2. Задаётся корреляционная матрица; перед расчётом она проверяется на
   положительную определённость.
3. Запускаются две симуляции — с независимыми и с коррелированными рисками.
   Коррелированные факторы генерируются гауссовой копулой (разложение Холецкого).
4. На каждой из 10 000+ итераций разыгрываются значения факторов и считается NPV.
5. По распределению NPV рассчитываются риск-метрики (VaR, Expected Shortfall,
   вероятность убытка), строятся графики и формируется инвестиционный вывод.

Главная идея: перейти от вопроса «чему равна NPV проекта?» к вопросу «с какой
вероятностью проект создаст или разрушит стоимость?».

---

## Как объяснить проект преподавателю

Короткая логика устного объяснения:

1. Обычный (точечный) NPV даёт одно число и не показывает риск.
2. Метод Монте-Карло строит распределение из тысяч сценариев и позволяет
   измерить вероятность убытка, VaR и Expected Shortfall.
3. В девелопменте риски (затраты, задержки, цены, регулирование) наступают
   совместно, поэтому их нельзя считать независимыми.
4. Учёт корреляции через гауссову копулу утяжеляет левый хвост распределения:
   вероятность убытка растёт, VaR и ES ухудшаются — независимая модель занижает
   риск.
5. Код разделён на интерфейс (`app.py`), математику (`model.py`) и сервис
   (`utils.py`), что делает его тестируемым и расширяемым.

Подробные материалы для защиты находятся в файлах `EXAM_GUIDE_NotebookLM.md`,
`CODE_WALKTHROUGH.md`, `DEFENSE_SCRIPT.md`, `EXAM_QA.md` и `PROJECT_MAP.md`.
