# Техническое описание проекта

Документ описывает архитектуру приложения, назначение модулей, поток данных и
организацию расчётной логики.

---

## 1. Структура проекта

```
monte-carlo-developer-dashboard/
├── app.py                  # интерфейс Streamlit и визуализация
├── model.py                # расчётное ядро (NPV, симуляция, метрики)
├── utils.py                # форматирование, проверка данных, тексты
├── requirements.txt        # зависимости
├── README.md               # описание проекта
├── .streamlit/
│   └── config.toml         # настройки оформления
└── docs/                   # документация
```

Приложение построено по принципу разделения ответственности: пользовательский
интерфейс, расчётная логика и вспомогательные функции вынесены в отдельные
модули. Это упрощает сопровождение, тестирование и повторное использование
расчётного ядра независимо от интерфейса.

## 2. Назначение файлов

| Файл | Назначение | Основные элементы |
|---|---|---|
| `app.py` | Интерфейс Streamlit и визуализация | конфигурация страницы, поля ввода, построение графиков, отображение таблиц и результатов |
| `model.py` | Математика и симуляция | `ProjectParams`, `RiskParams`, `calculate_npv`, `generate_correlated_risks`, `run_both_models`, `calculate_risk_metrics`, `calculate_sensitivity` |
| `utils.py` | Форматирование, проверка, тексты | `validate_correlation_matrix`, функции форматирования, построение таблиц, формирование текстового вывода |
| `requirements.txt` | Зависимости | streamlit, numpy, pandas, scipy, plotly |
| `.streamlit/config.toml` | Оформление | тема, цвета, шрифт, параметры сервера |

## 3. Поток данных в приложении

```
Ввод параметров (app.py)
      │
      ▼
ProjectParams + RiskParams + корреляционная матрица
      │
      ▼
проверка матрицы (utils.validate_correlation_matrix)
      │
      ▼
симуляция (model.run_both_models)
      │
      ├── генерация факторов риска (model.generate_correlated_risks)
      ├── расчёт NPV (model.calculate_npv)
      ├── показатели риска (model.calculate_risk_metrics)
      └── чувствительность (model.calculate_sensitivity)
      │
      ▼
таблицы и текстовый вывод (utils)
      │
      ▼
визуализация и отображение (app.py)
```

Интерфейсный модуль собирает входные данные, передаёт их в расчётное ядро,
получает результат и отображает его. Расчётный модуль не зависит от интерфейса.

## 4. Расчётный модуль `model.py`

Модуль содержит всю вычислительную логику и не зависит от Streamlit.

Параметры проекта и рисков описаны структурами данных (`dataclass`)
`ProjectParams` и `RiskParams`, что группирует входные величины и задаёт значения
по умолчанию.

Основные функции:

- `calculate_base_npv(params)` — детерминированная NPV при нейтральных значениях
  факторов;
- `calculate_npv(...)` — векторизованный расчёт NPV для массива сценариев;
- `generate_correlated_risks(corr_matrix, risk, n, seed, use_correlation)` —
  генерация факторов риска методом гауссовой копулы с разложением Холецкого;
  при `use_correlation=False` факторы генерируются независимо;
- `run_monte_carlo(...)` — полная симуляция для одной конфигурации;
- `run_both_models(...)` — расчёт обеих моделей (независимой и коррелированной);
- `calculate_risk_metrics(npv, base_npv, alpha)` — показатели риска (среднее,
  медиана, стандартное отклонение, перцентили, VaR, Expected Shortfall,
  вероятность отрицательного NPV);
- `calculate_sensitivity(factors, npv)` — коэффициенты чувствительности NPV к
  факторам.

Вспомогательные функции `_triangular_ppf` и `_discrete_ppf` реализуют обратные
функции предельных распределений. Функция стандартного нормального распределения
использует SciPy при его наличии и имеет реализацию на NumPy в качестве
резервной, что обеспечивает работоспособность модуля без SciPy.

## 5. Интерфейсный модуль `app.py`

Модуль реализует пользовательский интерфейс и визуализацию средствами Streamlit
и Plotly. Он включает конфигурацию страницы, боковую панель с базовыми
параметрами проекта и параметрами симуляции, поля ввода параметров факторов
риска, редактируемую корреляционную матрицу, запуск симуляции и отображение
результатов.

Интерфейс организован в виде последовательных разделов: описание модели, ввод
параметров, настройка факторов риска, корреляционная матрица, запуск симуляции,
результаты, сравнение моделей, чувствительность, итоговый вывод и методология.
Результаты симуляции сохраняются в состоянии сессии (`st.session_state`), что
исключает повторный пересчёт при перерисовке интерфейса.

## 6. Вспомогательные функции `utils.py`

Модуль обслуживает интерфейс и проверку данных. Он содержит функции
форматирования чисел и процентов в едином виде, функции построения таблиц
(сравнение показателей, чувствительность, корреляционная матрица), функцию
формирования текстового вывода по результатам расчёта, а также функции проверки
и коррекции корреляционной матрицы. Вынесение этих функций в отдельный модуль
разгружает интерфейсный и расчётный модули и позволяет тестировать их отдельно.

## 7. Визуализации

Графики строятся средствами Plotly:

- **Гистограмма NPV** — распределение исходов с отметками NPV = 0, среднего и
  VaR;
- **Кумулятивная функция распределения (CDF)** — вероятность того, что NPV не
  превысит заданный уровень, для обеих моделей;
- **Совмещённая гистограмма** — сравнение распределений независимой и
  коррелированной моделей;
- **Тепловая карта корреляций** — структура попарных корреляций факторов;
- **Диаграмма чувствительности** — ранжирование факторов по силе влияния на NPV.

## 8. Проверка корректности входных данных

Корреляционная матрица, введённая пользователем, проверяется функцией
`validate_correlation_matrix`. Проверяются форма, диапазон значений, симметрия,
единичная диагональ и положительная определённость (через разложение Холецкого).
При некорректной матрице строится ближайшая корректная (через коррекцию
собственных значений и нормировку диагонали), и пользователю выводится
соответствующее уведомление. Это гарантирует математическую корректность
последующей симуляции.

## 9. Воспроизводимость расчётов

Генерация случайных чисел использует фиксируемое начальное значение (seed),
задаваемое пользователем. При одинаковых входных параметрах и seed результаты
расчёта полностью воспроизводимы. Для разделения выборок независимой и
коррелированной моделей используется смещённое значение seed.
